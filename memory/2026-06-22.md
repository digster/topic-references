# 2026-06-22 — Project bootstrap

## What was built
Scaffolded **Topic References**, a no-build static study-resources site. Given a topic, the site
shows a short description followed by curated, verified resources (books, papers, articles, videos,
courses, tools, further reading).

### Files created
- `index.html` — landing page: hero, search box, alphabetical card grid, inline vanilla-JS filter
  (filters static DOM by card text + `data-tags`; works on `file://` and GitHub Pages).
- `styles.css` — single shared stylesheet; light/dark via `prefers-color-scheme`; responsive card
  grid; resource rows; level pills (`data-level`).
- `templates/topic-template.html` — commented skeleton for new topic pages.
- `topics/transformers.html` — seeded sample topic (Transformers), links researched & verified.
- `CLAUDE.md` — the workflow contract for adding topics (the key deliverable).
- `README.md`, `ARCHITECTURE.md`, `PROMPT.md`, `LICENSE` (MIT/digster), `.gitignore`, `.nojekyll`.
- `tests/check_site.py` — dev-only stdlib validator (`uv run tests/check_site.py`).

## Key decisions
- **No build step** is a hard constraint; site must work by double-clicking `index.html`.
- **Static links** in `index.html` instead of a `topics.json` + `fetch` (fetch is blocked on
  `file://`). Adding a topic edits `index.html` directly.
- **Sourcing:** live web search to find/verify real links + model knowledge for canonical sources
  (user's choice: options 1 + 3).
- **Landing page:** cards + client-side search filter (user's choice).

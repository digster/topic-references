# 2026-06-26 — Re-theme the site to "Brutalist Mono"

## Request
> Update the homepage to this theme. (Reference mockup provided: monospace UI type,
> yellow header band, heavy black display headings, thick black borders, hard offset
> drop shadows, warm-gray page framing a white panel, cobalt-blue accent.)

## Decisions (confirmed with the user)
- **Scope = whole site**, not homepage-only. `styles.css` is shared and the
  `.site-header` / `.site-footer` / `.container` / base `body` are common surfaces,
  so theming only the homepage would make the shared header look different between
  pages. Whole-site keeps it consistent.
- **Fixed light** look — no dark-mode variant. The brutalist theme is a single
  intentional light statement.

## What changed
Only `styles.css` was edited. No HTML, no JS, no template changes; the landing-page
search `<script>` is untouched (it reads `.topic-card`/IDs generically).

- **Tokens (`:root`)** remapped: warm-gray `--bg #eceae6`, white `--bg-elev`,
  near-black ink `--text/--border #111`, cobalt `--accent #2f56d8`, new `--yellow
  #f6d645`, hard offset shadows (`--shadow: 6px 6px 0`, `--shadow-hover: 3px 3px 0`),
  `--radius: 0`, `--frame: 3px`, plus system-stack `--font-mono` and `--font-display`
  (Golden Rule: no web fonts/CDNs).
- **Removed** the entire `@media (prefers-color-scheme: dark)` block (fixed-light).
- **Structural rules:** `<body>` is the frame (white panel, 3px black border, margin
  on warm-gray `html`); yellow header band with black bottom border; mono body; heavy
  uppercase system-sans display headings; square black-bordered search with hard
  shadow that "presses" on focus; blue uppercase result count (CSS-only — JS still
  outputs lowercase text); cards with hard offset shadows + blue-fill/white-text
  hover that slides into the shadow; squared black-bordered level pills; yellow
  prereq box; thick black section-heading underlines; framed footer.
